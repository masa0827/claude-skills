# storyboard-creator — 動画ラフ絵コンテ生成スキル

## 概要

動画の企画テキストを受け取り、**N×Nグリッドのラフ絵コンテ画像**を生成するための「画像生成プロンプト」を組み立てる。画像生成そのものの実行は、利用者の環境（OpenAI API / ChatGPT UI / Gemini API / ブラウザ）に合わせて分岐する。

目的は「動画の企画合意を秒で取る」ための素材作り。きれいな絵よりも、**構成・秒数・セリフ・カメラワークが一発で伝わる**ことが重要。

## なぜ GPT Image 2 を推奨するか

| AI | 強み | 弱み | 使い分け |
|----|------|------|---------|
| GPT Image 2 | 文字描画が強い。セリフ・カメラ指示をコマ内に焼き込める | キャラ一貫性はそこそこ | **第一選択**（ラフ絵コンテ用途） |
| Nano Banana Pro | キャラ・画風の一貫性が強い | 文字描画は弱め | キャラ固定の連続コマが必要な場合 |
| Nano Banana 2 | コスト低、一貫性○ | Pro より推論弱い | フォールバック |

文字焼き込みが弱いAIを使う場合は、コマ枠内のテキスト指示を削り、別途Markdownのコマ割り表でテキスト情報を補う運用に切り替える。

## 入力

### 必須
- 動画の目的（何を達成したいか）
- ターゲット（誰に見せるか）
- コアメッセージ（一言で何を伝えるか）
- 尺（秒数）
- アスペクト比（9:16 / 16:9 / 1:1）

### 任意
- トーン（明るい / 上品 / 勢い重視 など）
- キャラクター・被写体情報
- 参照画像URL
- 想定シーン骨子（空欄なら自動生成）

未入力の必須項目があれば `AskUserQuestion` で補完する。

## コマ数計算テーブル

| 動画尺 | コマ数 | グリッド候補 |
|-------|--------|-------------|
| 〜15秒 | 6 | 2×3 / 3×2 |
| 〜30秒 | 9 | 3×3 |
| 〜60秒 | 12 | 3×4 / 4×3 |
| 60秒以上 | 15〜16 | 4×4 / 3×5 / 5×3 |

アスペクト比と被写体の動きから、上記候補から1つ選ぶ。

## 実行手順

1. ユーザーから必須項目を取得
2. 尺からコマ数とグリッド形を決定
3. コマ単位でシーン設計（カット番号・秒数・シーン内容・セリフ・カメラ・モーション）
4. 画像生成プロンプトを組み立て（下記テンプレート）
5. **コマ割り表**（Markdownテーブル）と**画像生成プロンプト**（コードブロック）をユーザーに提示
6. ユーザー承認後、利用者の画像生成環境に応じて実行：
   - OpenAI API がある → API呼び出しコマンド例を提示（モデル名・エンドポイントはユーザーに確認）
   - ChatGPT UI で使う → プロンプトをクリップボードコピー推奨
   - Nano Banana Pro を使う → 文字焼き込み指示を弱め、画風指示を強める

## プロンプトテンプレート

画像生成AIに投げる最終プロンプトは以下の形式で出力する。`{}` の中身を企画に合わせて埋める。

```
【動画企画】
- 目的: {目的}
- ターゲット: {ターゲット}
- コアメッセージ: {コアメッセージ}
- 尺: {秒数}秒
- アスペクト比: {アスペクト比}
- トーン: {トーン}

【絵コンテ指示】
{グリッド形（例: 3×3）} のストーリーボード画像を1枚生成してください。
各コマは手描き風のラフな線画（pencil sketch / rough storyboard style）で、
背景は白またはオフホワイト、コマ枠を黒線で明確に区切ってください。
各コマの左上に「#カット番号 | {秒数}s」、下部中央に「{セリフ}」、
右上に「{カメラ指示}」を日本語で明瞭に文字入れしてください。

【コマ定義】
#1 ({秒数}s): {シーン内容} / セリフ「{セリフ}」/ カメラ: {カメラ指示} / モーション: {モーション}
#2 ({秒数}s): ...
...
#N ({秒数}s): ...

【スタイル】
- 画風: monochrome rough pencil storyboard sketch, 手描き風
- 線: 太すぎず、ラフに。細部は描き込まない
- 人物: 顔の輪郭のみで表情が分かるレベル
- テキスト: 日本語で可読に焼き込み
- パネル: black rectangular borders, equal size, clear grid layout

【禁止事項】
- カラー着色（モノクロで）
- 写真風の精細描写
- パネル外への描き込み
- 英語テキスト（全て日本語）
```

## 出力フォーマット

ユーザーには以下の順で出力する：

### 1. コマ割り表

| # | 秒数 | シーン内容 | セリフ | カメラ | モーション |
|---|------|-----------|-------|--------|-----------|
| 1 | 3s | ... | ... | ... | ... |

### 2. 画像生成プロンプト（コードブロック）

上記テンプレートの `{}` を全て埋めたもの。

### 3. 次アクション案内

「このプロンプトを GPT Image 2 / ChatGPT の画像生成 / Nano Banana Pro 等に貼り付けて実行してください」と案内。

## API呼び出し方針

画像生成API（モデル名・エンドポイント・認証）は**利用者に都度確認する**。OpenAIの画像生成APIは更新頻度が高いため、スキル内に固定値を書かない。

実装する場合：
- APIキーは環境変数（例: `OPENAI_API_KEY`）で受け取る。コードにハードコードしない
- スクリプトを1本に集約（例: `generate-storyboard.cjs`）、引数で企画ファイルを受け取る
- スクリプト雛形を書く前に、利用者に現在のAPI叩き方（モデル名・エンドポイント・必須パラメータ）を確認する

## 承認ゲート

以下の3ポイントでユーザー承認を挟む：

1. **入力項目の確認** — 目的・ターゲット・尺などの認識合わせ
2. **コマ割り表の確認** — セリフ・カメラ指示の微調整が入りやすいフェーズ
3. **画像生成プロンプトの確認** — 生成実行前の最終確認
4. **生成画像の確認** — NGならコマ割り表に戻って修正

## よくある失敗と対処

| 症状 | 原因 | 対処 |
|------|------|------|
| 日本語文字が崩れる | 英語との混在 | 焼き込みテキストを全て日本語に統一 |
| グリッド枠が揃わない | 指示が曖昧 | `equal size, clear grid layout` を明記 |
| 人物が描き込まれすぎる | スタイル指示が弱い | `minimal detail, silhouette level` を追加 |
| セリフ位置がバラつく | 位置指定なし | `text at bottom of each panel` を明記 |
| カラーになる | 画風指示が弱い | `monochrome pencil sketch` を強調、カラー禁止を明記 |
| コマ数が合わない | コマ定義不整合 | 定義コマ数とグリッド数が一致するか確認 |

## 重要な制約

- **推測で企画を太らせない**。入力が空欄なら「空欄のまま汎用的に作る」旨を明記
- **コマ数とグリッド形を必ず整合**させる（3×3なら9コマ、3×4なら12コマ）
- **セリフは短く**（1コマ10〜15文字以内）。長文は複数コマに分ける
- **カメラ指示は1つに絞る**（引き/寄り/パン/ズームイン/ズームアウト/固定/手持ち揺れ/トラック から1つ、混在禁止）

## 後続への引き渡し

絵コンテ承認後、各コマを動画化する場合：

1. コマ割り表からシーン別に分解
2. シーン別に**静止画生成AI**（Nano Banana 2 等）で本番画像を生成
3. 本番画像を**動画生成AI**（Seedance 2.0 / Kling / Runway 等）に投入
4. 動画編集ツール（Remotion / CapCut / Premiere 等）で結合

この後続フローは別スキルに委ねる。**storyboard-creator の責任範囲は「絵コンテ合意まで」**。
