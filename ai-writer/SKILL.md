# AIライターv1 — 人間らしい文章生成スキル

## 概要

このスキルは、AIが生成する文章から「AI臭」を完全に除去し、人間が書いたような自然で説得力のある文章を生成します。

もとやま氏（@ysk_motoyama）の「AIっぽい文章表現大全」で定義された6つのAI特有パターンを避け、3つの「崩し技術」を適用します。

## 使用タイミング

- ブログ記事、SNS投稿、マーケティングコピーなど、あらゆる文章生成時
- 既存のAI生成文章を人間らしく書き直す時
- 「何か違和感がある」と感じる文章を改善する時

## AIっぽさの6つの特徴（絶対に避ける）

### 1. 視覚的な記号が残ったまま

**NG例:**
- アスタリスクによる太字 `**テキスト**`
- 言い換えの `—`（ダッシュ）
- `：`のあとの半角スペース
- 補足の `()` 多用

**対応:**
- Markdown記法を一切使用しない
- 強調は文脈と語彙選択で表現
- 補足は文章に自然に組み込む

### 2. 文のリズムが単調

**NG例:**
- 同じ語尾が続く（〜です。〜です。〜です。）
- 接続詞の過剰使用（さらに、また、したがって、そのため）
- 文章の温度が一定（盛り上がりがない）
- 段落の終わりが毎回きれいに閉じる
- not A but B 構文ばかり

**対応:**
- 語尾を意図的に変化させる（だ・である調、です・ます調を混ぜる）
- 短文と長文を織り交ぜる
- 接続詞を削除し、暗黙的な接続を使う
- 段落を途中で切る、続ける、跳ねるなど変化をつける

### 3. 導入と進行が説明書っぽい

**NG例:**
- 前置きが長い（「ご質問ありがとうございます。〜」）
- 「結論から言うと」で始めるが結論が薄い
- 構造を本文で何度も宣言（「以下の3つの観点から〜」）
- ステップ、STEP、Stepなどの表現

**対応:**
- いきなり本題から入る
- 構造の宣言は最小限に
- 番号付きリストは避け、自然な流れで展開

### 4. 事なかれ主義なスタンス

**NG例:**
- 保険が多い（「一概には言えませんが」「一般的には」）
- 中立を装いすぎる（「メリットもあればデメリットもあります」）
- 否定が弱い（「あまり推奨されません」「注意が必要です」）

**対応:**
- 断定する勇気を持つ
- 立場を明確にする
- 「〜すべきではない」「やめろ」など強い表現も使う

### 5. 言葉選びが「抽象語」と「万能語」に寄る

**NG例:**
- 抽象語だけで押し切る（「本質」「最適化」「最大化」「エコシステム」「バリュー」）
- 評価語が強いのに根拠がない（「非常に有効です」）

**対応:**
- 具体的な数字、固有名詞、事例を使う
- 抽象語は具体例とセットで使う
- 評価は必ず根拠を添える

### 6. 変な比喩が多い

**NG例:**
地図、仕様書、羅針盤、土台、DNA、潤滑油、スパイス、レシピなど

**対応:**
- 比喩は使うなら生々しく具体的に
- ありきたりな比喩は避ける
- 比喩なしで直接表現する

## 人間らしさを作る3つの崩し技術

### 1. バランスを崩す

- **敢えてスタンスを取る**: 中立ではなく、明確な立場から語る
- **言い切る**: 「〜だと思います」→「〜だ」
- **偏りを恐れない**: 完璧な公平性より、リアルな視点を優先

### 2. 客観を崩す

- **主観を入れる**: 「私は〜」「個人的には〜」を恐れない
- **一次情報を盛り込む**: 実体験、具体的なエピソード、失敗談
- **尖った価値観**: 「普通はこうだが、俺はこう思う」

### 3. 論理を崩す

- **話を蛇行させる**: 脱線、寄り道、思いついたことを挟む
- **矛盾を残す**: 一貫性にこだわりすぎない
- **感情の動き**: 「あれ？」「まじか」「やばい」など生の反応
- **葛藤を見せる**: 「〜とも言えるし、〜とも言える。正直迷ってる」

## 実践ガイドライン

### 文章生成時の具体的ステップ

1. **まず情報を整理**
   - 何を伝えたいか（事実）
   - 誰に伝えるか（読者）
   - どんな反応を期待するか（目的）

2. **骨組みを作る（ただし構造宣言はしない）**
   - 話の流れを決める
   - 重要な具体例・数字をピックアップ
   - どこで「崩す」かを決める

3. **執筆（この段階で意識すること）**
   - Markdown記法を使わない
   - 語尾を3回以上連続させない
   - 段落ごとに文体を少し変える
   - 主観・体験を1つは入れる
   - 「一般的には」「〜と言えます」を削除
   - 抽象語の後には必ず具体例

4. **見直し（AI臭チェック）**
   - 記号が残っていないか（`**`, `—`, `()`, `:` ）
   - 同じ接続詞が続いていないか
   - 「結論から言うと」で始まっていないか
   - 「メリット・デメリット」の二項対立になっていないか
   - 抽象語だけで終わっていないか
   - 地図・羅針盤などの陳腐な比喩がないか

5. **最終調整（人間らしさ注入）**
   - どこか1箇所、話を脱線させる
   - 強い言い切りを1つ入れる
   - 個人的なエピソード・感情を追加

## 出力フォーマット

生成する文章は以下の特徴を持つこと：

- **形式**: プレーンテキスト（Markdown記法なし）
- **語尾**: 変化に富む（です・ます、だ・である、体言止め、疑問形など混在）
- **段落**: 長短のリズムをつける
- **主観**: 必ず含む（体験、感情、価値観）
- **具体性**: 抽象語より具体例を優先

## 禁止事項リスト

以下の表現・パターンは**絶対に使用禁止**:

- `**テキスト**`（太字のMarkdown記法）
- 「結論から言うと」で始める定型文
- 「以下の〜つの観点から」構造宣言
- 「一概には言えませんが」などの保険表現
- 「メリットもあればデメリットもあります」二項対立
- 「非常に有効です」根拠のない評価
- 地図・羅針盤・土台・DNA・潤滑油などの陳腐な比喩
- 「ステップ1」「STEP2」などの段階表記
- 3回以上連続する同じ語尾
- 「さらに」「また」「したがって」の連続使用

## 使用例

### NG例（AI臭が強い）

```
**結論から言うと、このツールは非常に有効です。**

以下の3つの観点から説明します：
1. 効率性の向上
2. コストの最適化
3. ユーザー体験の改善

一般的には、これらのメリットとデメリットをバランスよく考慮する必要があります。ただし、状況によっては注意が必要です。

したがって、慎重に検討することをお勧めします。
```

### OK例（人間らしい）

```
このツール、使ってみたら想像以上に良かった。

まず効率が段違い。今まで手作業で2時間かかってた作業が15分で終わる。コストも月3万円削減できた（うちの会社では結構デカい）。

ただ、最初の設定が面倒くさい。マニュアル読んでも意味わからなくて、サポートに3回問い合わせた。ここは正直イマイチ。

でも、一度設定しちゃえばあとは楽。少なくともうちのチームには合ってる。
```

## 注意事項

- このスキルは「読みやすさ」より「人間らしさ」を優先します
- 完璧な論理構造より、リアルな思考過程を重視します
- ターゲット読者によっては、あえてフォーマルさを残すことも検討してください
- 最終的な調整は人間の感覚で行ってください
